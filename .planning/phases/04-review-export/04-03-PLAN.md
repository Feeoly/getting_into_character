<context>
- **导出：** 组装 markdown /纯文本（转写分段为主；可选一行会话元数据）；下载逻辑参考 **`saveRecordingToDisk.ts`**（`showSaveFilePicker` 优先）。文件名 `gic-review-{session}-{take}` 截断安全化。
- **删除 take：** `transcriptRepo` 新增 **`deleteTakeData(sessionId, takeId)`**：删该 take 的 jobs、segments、`pauseEvents` 中带该 `takeId` 的行。成功后 `router.push` 会话详情或首页（planner 定默认）。
- **删除会话：** `sessionRepo`（或专用模块）**事务**内删除：`sessions`、`rehearsalSettings`、`uploadedBackgrounds`（先取 settings 中 `uploadedBackgroundId`）、`pauseEvents`、`transcriptionJobs`、`transcriptSegments`。顺序避免外键式残留。
- **Chat UI：** 组件化 `ReviewChat`；`localStorage` 键记录 AI 同意；未同意禁用发送。消息区滚动到底；用户/助手气泡样式按 UI-SPEC。
- **API：** `POST` body：`{ messages: {role:'user'|'assistant'|'system', content:string}[], context?: { question?: string, answer?: string, transcriptExcerpt: string, pausesExcerpt?: string } }`（字段名可微调）。服务端 **注入 system**：说明仅中文面试教练、输出结构（要点/例句/引用转写或停顿）、禁止杜撰未给片段。`BAILIAN_BASE_URL` trim 末尾 `/`，请求 `${base}/chat/completions`（若百炼 path 不同，以环境变量整段 base 覆盖，文档注释说明）。**不设 Key 时返回 503 + JSON message**。
- **体积：** 对 `messages`/`context` 总字符设上限（如 ~32k 字符），超限400。
</context>
