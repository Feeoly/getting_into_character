<threat_model>
## Trust Boundaries

| Boundary | Description |
|---------|-------------|
| 用户交互 → 状态更新 | “开始/结束/重录”会改变本地持久化状态，需要防止非法状态转换与错误不透明 |
| 路由参数 → 数据读取 | `/session/[id]` 的 `id` 来自 URL（不可信），需要健壮处理不存在/格式异常 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|----------|----------|-----------|-------------|-----------------|
| T-01-04 | Tampering | `updateSessionStatus` 状态机 | mitigate | 在 Repo 层做状态转换校验（拒绝非法回退/跳转），并返回可显示错误 |
| T-01-05 | Denial of Service | IndexedDB 错误导致状态无法落盘 | mitigate | 捕获异常，UI 显示 UI-SPEC Error state，并建议刷新/检查存储权限或空间 |
| T-01-06 | Information Disclosure | 误引入联网依赖影响信任 | mitigate | 仍保持无第三方脚本/外链资源；断网路径覆盖 `/session/new` 与 `/session/[id]` |
</threat_model>
