<threat_model>
## Trust Boundaries

| Boundary | Description |
|---------|-------------|
| 用户输入 → 本地存储 | 会话名称/备注等文本写入 IndexedDB（需要校验与错误处理） |
| 应用运行时 → 网络 | Phase 1 需避免任何非必要外链请求以支撑“默认不上传”的可验证性 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | 首页加载资源/脚本 | mitigate | 禁用第三方脚本与外链字体；使用 system-ui；通过断网可运行路径验证（与 UI-SPEC/Pitfall 对齐） |
| T-01-02 | Denial of Service | IndexedDB 写入/空间不足 | mitigate | 统一在 Repo 层捕获异常并返回可显示错误；UI 使用 UI-SPEC Error state 文案指引检查存储权限/空间 |
| T-01-03 | Tampering | 本地持久化数据结构变更 | mitigate | 用 Zod schema 校验写入与读取（必要时做容错/迁移策略），避免脏数据导致崩溃 |
</threat_model>
