  <action>
    - 在 `pickBestMimeType()` 中按优先级探测（示例优先音频）：
      - audio/webm;codecs=opus
      - audio/webm
      - audio/mp4（若支持）
      - 回退：不传 options（让浏览器选择）
    - `startRecording()`：
      - 调用 `navigator.mediaDevices.getUserMedia()`，参数按 cameraEnabled 选择 `{ audio:true }` 或 `{ audio:true, video:true }`
      - 用 timeslice（例如 1000ms）启动 recorder，持续收集 `dataavailable` chunks
      - 暴露当前 live `MediaStream`（用于预览窗显示视频，且不影响录制）
    - `stopRecording()`：
      - 结束录制后，把 chunks 合并成 Blob，创建 object URL 用于回放
      - 返回 recorder 实际 `mimeType`（用于 UI 文案/兼容处理）
    - 错误处理：把 NotAllowedError/NotFoundError/NotSupportedError 转成可显示的错误对象（不吞掉）。

<threat_model>
## Trust Boundaries

| Boundary | Description |
|---------|-------------|
| 用户点击 → 浏览器权限系统 | 开始录制触发麦克风/摄像头权限请求（必须渐进授权） |
| 媒体流 → 回放 UI | Blob/object URL 用于回放，需避免泄漏与资源不释放 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|----------|----------|-----------|-------------|-----------------|
| T-02-04 | Repudiation / Social | 权限请求时机与录制状态可见性 | mitigate | 仅在点击开始/开启摄像头后请求权限（D-02）；UI 明确展示录制状态与错误原因 |
| T-02-05 | Denial of Service | 录制 Blob/URL 占用内存 | mitigate | timeslice 分片收集；停止后释放 stream tracks；替换回放时 revoke 旧 object URL |
| T-02-06 | Tampering | mimeType 不兼容导致回放失败 | mitigate | `isTypeSupported()` 运行时探测；使用 recorder 实际 mimeType 记录并按 kind 选择 audio/video 回放控件 |
</threat_model>
