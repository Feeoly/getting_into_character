      - 在开始录制的用户手势之后创建/`resume()` AudioContext（避免被挂起）

<threat_model>
## Trust Boundaries

| Boundary | Description |
|---------|-------------|
| 媒体流 → 分析管线 | Web Audio 对媒体流做实时分析，需避免卡顿与资源泄漏 |
| 事件记录 → 本地存储 | 高频事件写入可能导致存储压力与性能抖动（需节流/冷却） |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|----------|----------|-----------|-------------|-----------------|
| T-02-07 | Denial of Service | pause detector 采样循环 | mitigate | 只在录制中运行；停止时取消 rAF；使用 cooldown；避免每帧写入 DB（只在 pause end 写入一次） |
| T-02-08 | Tampering | settings 阈值过大/过小导致误报 | mitigate | 阈值用 Zod 做范围限制；检测器对参数做防御性 clamp；提示与事件字段记录 threshold_ms 便于复盘解释 |
| T-02-09 | Information Disclosure | pause events 内容 | accept | 事件只记录时间/阈值/状态，不包含音频内容与文本（与 D-11 对齐），隐私风险低且本地存储 |
</threat_model>
