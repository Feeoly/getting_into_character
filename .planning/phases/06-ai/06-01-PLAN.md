<objective>
落实 **06-RESEARCH §3～4、§6**：新增 **`POST /api/role/enhance`**（与 `/api/review/chat` 解耦）；抽取或复用百炼 **`fetch` 骨架**（503/502、JSON 非流式返回）。扩展 **`SESSION_SCHEMA` / `Session`**：并存 **`roleCardAiText`**、**`roleCardAiUpdatedAt`**、**`roleCardPreferAi`**（`boolean`，默认 `true`：存在 AI 稿时优先展示 AI；`false` 时展示本地 `roleCardText`）。不改动 Dexie `version(1)` 表结构（仅对象新键）。导出纯函数 **`getEffectiveRoleCardText(session): string | undefined`**：若 `roleCardPreferAi !== false` 且 `roleCardAiText` 有值则返回其 trim，否则返回 `roleCardText`。
</objective>

<context>
- **请求体 Zod 示例字段名（可微调须全链路一致）：** `draftText`（必填，`max(8000)`）、`moodPreset`/`moodCustom`/`trigger`（可选短字符串，仅作模型附加上下文）。
- **响应 JSON：** `{ content: string }`，`content` 为模型全文；前端负责再跑 Zod `roleCardText` 同级校验后再写入 IndexedDB。
- **MAX_BODY_CHARS：** 与 `review/chat` 同量级（如 32000）或更严；整包超限返回 400。
- **system prompt 必须包含字面标题：** `【状态描述】`、`【可执行表达指令】`、`【禁忌】` — 与 `src/app/session/[id]/_lib/roleCopy.ts` 中 **`roleSectionTitles`** 一致（grep 验收）。
</context>

| Boundary | Description |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01-01 | I | 超长正文 DoS | mitigate | Zod max + MAX_BODY_CHARS |
| T-06-01-02 | I | Prompt 注入 | mitigate | system 固定；user 仅拼接标注字段 |
| T-06-01-03 | E | API Key 泄露 | mitigate | 仅服务端 `process.env`；不返回客户端 |
| T-06-01-04 | S | 模型输出有害内容 | mitigate | system 禁忌段要求；入库前长度截断/校验 |
</threat_model>
