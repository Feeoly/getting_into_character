<objective>
在 **Dedicated Worker** 中集成 **`@huggingface/transformers`** 的 `automatic-speech-recognition`（量化多语 Whisper，与 RESEARCH 一致），主线程 **`transcriptionRunner`** 负责入队、拉取/解码音频、与 Worker 通信、将 **`TranscriptChunk`** 写入 Dexie 并驱动任务状态。排练页在 **录制成功结束** 时自动入队（D-04）；**`takeId`** 在 `stopRecording()` 成功路径生成并随 `StopRecordingResult` 返回；**Blob 先入 IndexedDB** 再安全管理 `blob:` URL 生命周期（对齐 RESEARCH「勿过早 revoke」）。**默认引擎不得调用公网 STT API**；仅允许 Hub/自托管 **静态模型文件** HTTPS 下载（STT-03）。
</objective>

<context>
- 决策：D-01（浏览器内 WASM/ONNX）、D-02（实现真实 `TranscriptionEngine`）、D-04、D-08（句级/短分段，`return_timestamps` 或后处理切分 ≤10s 窗口）、D-09、D-10。
- 产品边界（RESEARCH Open Q1）：**v1 转写对象 = 用户录制生成的 Blob（混合轨）**；不在此 plan 拆分「仅麦克风轨」。
</context>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| 用户 Blob | 视为用户自有数据；解码前不执行脚本内容 |
| Worker ← → Hub CDN | 仅加载模型权重；须 HTTPS；校验由浏览器 + 包管理版本锁定 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02-01 | S | 依赖/模型 CDN | mitigate | 锁定 `@huggingface/transformers` 与 **固定 `modelId` 常量**；文档注明供应链风险与版本冻结；禁止 `eval`/动态脚本加载用户输入 |
| T-03-02-02 | D | 恶意超大 Blob | mitigate | 入队前检查 `blob.size` 上限（如 ≤ 200MB 可配置常量），超限则 `markFailed` + 错误码 |
| T-03-02-03 | I | `postMessage` 载荷 | mitigate | 仅允许结构化 clone 的 `ArrayBuffer`/元数据；不传递 `Function` 或任意字符串作为 code |
| T-03-02-04 | E | Worker OOM | accept | 失败 toast 由 03-03 消费状态；本 plan `markFailed` |
</threat_model>
